# Role Onboarding Checklist

## Purpose
This checklist helps new team members quickly understand their role, responsibilities, and key touchpoints with other team members on OctoAcme projects.

## How to Use
- Copy this template for each new team member
- Customize based on their specific role
- Complete checklist items during first 1-2 weeks
- Review with Project Manager or team lead

---

## General Onboarding (All Roles)

### Week 1: Foundation
- [ ] Review [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Read [Roles and Personas](octoacme-roles-and-personas.md) document
- [ ] Access project repository and documentation
- [ ] Join relevant communication channels (Slack, Teams, etc.)
- [ ] Attend team standup or sync meeting
- [ ] Review current sprint/iteration goals
- [ ] Introduction meeting with Project Manager
- [ ] Review project charter and success metrics

### Week 1-2: Role Integration
- [ ] Shadow key meetings relevant to your role
- [ ] Review role-specific artifacts and deliverables
- [ ] Meet with direct collaborators (see Key Interactions in your role description)
- [ ] Understand current project phase and priorities
- [ ] Review past retrospectives to learn from team history

### Ongoing
- [ ] Set up 1:1 meetings with key stakeholders
- [ ] Contribute to first team deliverable
- [ ] Provide onboarding feedback for process improvement

---

## Role-Specific Onboarding

### For Product Owners
- [ ] Review current product backlog and priorities
- [ ] Meet with stakeholders to understand business context
- [ ] Review past user research and customer feedback
- [ ] Understand acceptance criteria standards
- [ ] Review product metrics and success indicators
- [ ] Schedule regular sync with Product Manager and Project Manager
- [ ] Participate in backlog refinement session

### For Quality Assurance Leads
- [ ] Review existing test strategies and plans
- [ ] Access testing environments and tools
- [ ] Review CI/CD pipeline and automated test coverage
- [ ] Meet with Developers to understand codebase and testing practices
- [ ] Review defect tracking process and current issues
- [ ] Understand quality metrics and benchmarks
- [ ] Review Definition of Done and acceptance criteria standards

### For Business Analysts
- [ ] Review existing requirements documentation
- [ ] Meet with stakeholders to understand business objectives
- [ ] Understand requirements elicitation process
- [ ] Review user story and acceptance criteria templates
- [ ] Meet with Product Owner to align on backlog priorities
- [ ] Understand stakeholder communication cadence
- [ ] Review past requirements-related issues or gaps

### For Change Managers
- [ ] Review project scope and anticipated organizational impact
- [ ] Meet with stakeholders to assess change readiness
- [ ] Review past change communications and training materials
- [ ] Understand adoption metrics and success criteria
- [ ] Develop initial change impact assessment
- [ ] Identify key users and champions for the change
- [ ] Review communication channels and stakeholder preferences

### For DevOps Specialists
- [ ] Access infrastructure and deployment tools
- [ ] Review CI/CD pipeline configuration
- [ ] Understand deployment process and schedules
- [ ] Review monitoring and alerting setup
- [ ] Meet with Developers to understand build requirements
- [ ] Review incident response procedures
- [ ] Understand security and compliance requirements
- [ ] Access production support documentation

### For UX/UI Designers
- [ ] Review existing design systems and brand guidelines
- [ ] Access design tools and repositories
- [ ] Review past user research findings
- [ ] Meet with Product Owner to understand user needs
- [ ] Review current wireframes and prototypes
- [ ] Understand usability testing process
- [ ] Meet with Developers to understand technical constraints
- [ ] Review accessibility standards and requirements

### For Developers
- [ ] Set up local development environment
- [ ] Review codebase architecture and coding standards
- [ ] Access development tools and CI/CD pipeline
- [ ] Review PR workflow and code review process
- [ ] Meet with DevOps Specialist about deployment process
- [ ] Review technical documentation
- [ ] Complete first code contribution (bug fix or small feature)
- [ ] Understand branching strategy and release process

### For Project Managers
- [ ] Review all project artifacts (charter, plans, risk register)
- [ ] Meet individually with each core team member
- [ ] Understand stakeholder landscape and communication preferences
- [ ] Review project board and tracking tools
- [ ] Understand escalation paths and decision-making authority
- [ ] Review budget and resource constraints
- [ ] Access historical project data and past retrospectives
- [ ] Schedule key recurring meetings (standups, syncs, reviews)

---

## Completion

- [ ] Onboarding checklist reviewed with team lead or Project Manager
- [ ] Feedback provided on onboarding experience
- [ ] Role responsibilities and expectations clearly understood
- [ ] Key touchpoints and collaboration patterns established
